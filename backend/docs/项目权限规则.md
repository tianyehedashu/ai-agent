# 项目权限规则

> 本文说明**谁能看到哪些页面、能使用哪些功能**，以及**不同人看到的数据是否相同**（产品 / 运营 / 测试可读）。  
> 技术实现见 [PERMISSION_SYSTEM_ARCHITECTURE.md](PERMISSION_SYSTEM_ARCHITECTURE.md) 与 [CODE_STANDARDS.md](CODE_STANDARDS.md)。

---

## 1. 角色说明

系统用三层身份组合判断权限，不要只记「是否登录」。

| 维度 | 说明 | 典型取值 |
|------|------|----------|
| **登录态** | 是否完成注册登录 | 匿名访客 / 已登录用户 |
| **平台角色** | 账号在整站的身份 | 普通用户、只读用户（预留）、**平台管理员** |
| **团队角色** | 在**当前选中的团队**里的身份 | 所有者、管理员、成员；个人空间一般为「所有者」 |

**团队内权限（AI 网关控制台）**

- **所有者 / 团队管理员**：可改团队凭据、模型、路由、预算、告警、成员等（「可管理」）。
- **成员**：可查看团队配置与日志，**不能**改团队级配置；可**创建并仅管理自己的**虚拟 Key（见 §2.2）。
- **平台管理员**：在整站额外拥有运维类菜单与跨团队统计；在某一共享团队里是否可改配置，仍看该团队内的角色（平台身份不自动等于团队所有者）。

## 2. 数据可见性（不同的人看到的数据是否一样）

**结论：不一样。** 系统按「你是谁、属于哪些团队、当前选中的是哪个团队」过滤列表与详情；没有「全站所有人看到同一份业务数据」的设计（平台管理员在部分运维场景除外，见 §7）。

### 2.1 三条原则

1. **工作区（团队）是数据边界**  
   会话、Agent、大部分网关配置、调用日志等都挂在某个工作区下。用户**只能访问自己作为成员**的工作区里的数据；未加入的工作区，列表为空或访问被拒绝。

2. **个人空间也是独立工作区**  
   注册后自动有一个「个人团队」。个人凭据、个人模型挂在个人团队下，**其他用户看不到**；与后来加入的「共享团队」互不混用，除非在界面里切换到那个团队。

3. **当前选中的团队决定网关里「团队/共享」看到什么**  
   在 AI 网关顶部切换团队后，凭据、模型、路由、日志、预算等**团队维度**列表会随之切换；选团队 A 时看不到团队 B 的配置。

### 2.2 按数据类型：谁能看到什么

| 数据类型 | 可见范围 | 补充说明 |
|----------|----------|----------|
| **聊天会话与消息** | 本人可访问工作区内的会话 | 匿名访客仅能看到与本访客身份绑定的会话，看不到注册用户或其他访客的数据 |
| **Agent** | 默认仅本工作区成员 | 若 Agent 设为**公开**，同工作区内其他成员可**查看**该 Agent，仍不能代为修改；列表接口仍主要展示本工作区内的 Agent |
| **网关 · 个人凭据 / 个人模型** | 仅本人（个人工作区） | 与「团队/共享」页签无关；他人无法看到你的 BYOK 账号与个人模型别名 |
| **网关 · 团队凭据 / 模型 / 路由 / 预算 / 告警** | 当前选中团队 + 成员身份 | 须为该团队成员；列表中会合并展示**系统级**全局模型/路由（全员可见的配置），与当前团队自有配置区分展示 |
| **网关 · 虚拟 Key** | **仅创建者本人** | 每人只能看到、揭示明文、吊销**自己创建**的 Key；团队所有者/管理员**不能**查看或代用成员的 Key（与团队共享「凭据」不同） |
| **网关 · 调用日志** | 当前团队的日志 | 非成员看不到该团队日志 |
| **设置 · 平台 API 密钥** | 仅本人 | 与他人无关 |
| **MCP 服务器（用户自建）** | 仅本人 | 每人维护自己的连接配置 |
| **视频 / Listing 任务** | 本人工作区内 | 与当前账号及工作区绑定，看不到他人任务 |
| **网关 · 平台统计** | 仅平台管理员 | 跨工作区汇总的运维统计，普通用户无此菜单 |
| **对象存储配置** | 全站一份配置 | 仅平台管理员可查看与修改，影响 Listing 等功能的存图方式 |

### 2.3 常见对比（便于测试与客服）

| 场景 | 用户 A 能看到 | 用户 B 能看到 | 是否相同 |
|------|----------------|----------------|:--------:|
| A、B 无共同工作区 | 各自个人空间的数据 | 各自个人空间的数据 | 否（互不可见对方业务数据） |
| A、B 同在共享团队 T | 个人空间 + 团队 T 的网关团队数据 | 个人空间 + 团队 T 的网关团队数据 | 部分相同（仅 T 相关配置与日志一致；会话、Agent 仍各属各工作区） |
| B 在团队 T1、T2，网关切到 T1 | — | T1 的团队凭据/日志等 | 切换团队前后列表**会变化** |
| B 为团队 T 的**成员**（非管理员） | — | T 的配置只读；虚拟 Key **仅 B 自己创建的** | A（所有者）**看不到** B 的虚拟 Key |
| 平台管理员 vs 普通用户 | 个人 + 已加入团队 + 运维/统计 | 个人 + 已加入团队 | 管理员多运维类数据；**不应**假设能随意浏览任意租户的全部会话（策略待产品定稿） |

### 2.4 与「菜单权限」的关系

- **菜单/按钮**：决定你能不能进入某功能、能不能点「保存」「删除」。  
- **数据范围**：决定进入后列表里**有哪些行**、点开详情是否报错。  

例如：两人都能看到「AI 网关 → 调用日志」，但若不在同一团队，或选中的团队不同，**日志内容不同**；虚拟 Key 列表则**按人隔离**（每人只看到自己的 Key）。

## 4. 匿名访客

**可以使用**

- 聊天、会话列表与历史
- Agents 列表（受本访客数据范围限制）
- 用户 MCP 配置、系统 MCP（查看与复制接入说明）
- 视频任务、Listing 创作
- 设置中的通用项（如主题）

**不能使用或受限**

- 侧栏不出现 **AI 网关**（须登录）
- 设置中 **API 密钥、账户信息** 的保存与删除账户
- 仅能访问与本访客会话相关的数据，不能访问其他用户或团队的数据

## 4. 已登录普通用户

注册登录后默认拥有 **个人团队（工作区）**，在个人团队内一般为 **所有者**，对「个人」类 Gateway 资源可完全自助管理。

### 5.1 设置

| 页签 | 功能 |
|------|------|
| 通用 | 主题、通知等 |
| API 密钥 | 创建、查看、吊销**平台 API 密钥**（用于调用产品 API；与网关内的「虚拟 Key」不是同一类） |
| MCP 服务器 | 管理自己的 MCP 连接配置 |
| 账户 | 修改昵称、厂商用户 ID 等 |

说明：历史上「凭据」「个人模型」曾放在设置里，现已迁到 **AI 网关 → 凭据 / 模型** 的「个人」页签；旧链接会自动跳转到网关对应页面。

### 5.2 AI 网关（须登录）

进入 **AI 网关** 后，左侧为二级菜单。下表「可管理」= 当前团队为所有者或团队管理员，或账号为平台管理员。

| 子菜单 | 所有人 | 可管理 | 仅平台管理员 |
|--------|:------:|:------:|:------------:|
| 概览 | 查看仪表盘 | — | — |
| 调用指南 | 阅读文档 | — | — |
| 虚拟 Key | 仅查看**自己创建**的 Key | 任意团队成员可**新建**；仅创建者可吊销自己的 Key | — |
| 凭据 | 查看个人 + 当前团队 | 个人：增删改、探测、导入模型；团队：增删改团队凭据 | 可编辑**系统级**凭据 |
| 模型 | 查看个人 + 团队模型 | 注册、编辑、删除、连通性测试 | — |
| 定价目录 | 「我的价格」 | 「下游售价」（团队管理员及以上） | 「上游成本」 |
| 虚拟路由 | 查看 | 增删改 | — |
| 预算配额 | 查看 | 创建、调整 | — |
| 调用日志 | 查看本团队日志 | — | — |
| 告警规则 | 查看 | 增删改 | — |
| 团队成员 | 查看团队与成员 | 新建团队、添加/移除成员；所有者可删除非个人团队 | 亦可删除团队（与所有者相同） |
| 平台统计 | — | — | 查看（非管理员打开会回到概览） |

**凭据页 — 两个页签**

| 页签 | 普通用户能做什么 |
|------|------------------|
| **个人** | 添加自己的上游账号（BYOK）、编辑、删除、测试连接、从凭据一键导入**个人模型**；与个人团队绑定，不依赖切换团队 |
| **团队 / 共享** | 查看当前选中团队下的团队凭据；个人团队所有者 / 团队管理员可创建、修改、删除团队凭据 |

**模型页 — 两个页签**

| 页签 | 普通用户能做什么 |
|------|------------------|
| **个人** | 在个人凭据上注册对外模型别名、测试、删除 |
| **团队 / 共享** | 查看团队模型；团队管理员及以上可从目录注册并维护团队模型 |

**聊天里选模型**：列表来自「系统可见模型 + 个人已注册模型」，与网关控制台里的「团队模型列表」用途不同。

**定价目录 — 三个页签**

| 页签 | 谁看得见 | 谁能改 |
|------|----------|--------|
| 我的价格 | 登录用户 | 按团队定价规则（通常需团队管理员） |
| 下游售价 | 团队管理员及以上 | 同上 |
| 上游成本 | 仅平台管理员 | 平台管理员 |

### 5.3 其他功能模块

| 模块 | 已登录普通用户 |
|------|----------------|
| **Agents** | 在本工作区内创建、编辑、删除 Agent |
| **聊天** | 建会话、对话；可选用系统模型或个人模型 |
| **MCP 服务器** | 管理自己的 MCP 配置 |
| **系统 MCP** | 查看接入说明；**动态工具与提示词管理仅平台管理员** |
| **视频 / Listing** | 在本工作区内创建与管理任务 |

## 6. 共享团队中的「成员」

用户被邀请进**共享团队**且角色为 **成员** 时，在网关里切换到该团队后：

| 能力 | 成员 | 团队管理员 / 所有者 |
|------|:----:|:------------------:|
| 查看概览、指南、日志、模型、路由、凭据、预算、告警 | 是 | 是 |
| 修改团队凭据、团队模型、路由、预算、告警 | 否 | 是 |
| 虚拟 Key | 仅自己创建、自己使用 | 仅自己创建、自己使用 |
| 定价「下游售价」 | 不可见 | 可见可改 |
| 添加/移除团队成员、新建共享团队 | 否 | 是 |

## 7. 平台管理员

在**已登录普通用户的全部能力**之外，额外包括：

| 类别 | 功能 |
|------|------|
| **平台运维** | 侧栏 **对象存储**：配置 Listing 等功能的图片存储（本地 / 对象存储） |
| **网关跨团队** | **平台统计**：按凭据维度查看全局调用统计 |
| **网关配置** | 编辑 **系统级** 上游凭据；可从配置同步全局模型目录（运维操作，控制台未必有按钮） |
| **定价** | 查看与维护 **上游成本**；可维护全局范围定价 |
| **系统 MCP** | 管理动态工具与提示词（普通用户仅可复制接入配置） |

**说明**：平台管理员在数据层可能具备更广的读取能力，具体边界尚未最终定为产品策略；**业务功能不要设计成「管理员可看任意租户全部业务数据」**，详见架构文档中的待决事项。

平台管理员**不会自动**成为每个共享团队的所有者；在某个团队里能否改配置，仍取决于该团队内的角色。

---

## 7. 只读平台用户（viewer）

系统预留 **viewer** 平台角色，**当前界面与普通用户相同**，未单独隐藏写操作按钮。若产品需要「全站只读账号」，需另行定义菜单与按钮规则。
