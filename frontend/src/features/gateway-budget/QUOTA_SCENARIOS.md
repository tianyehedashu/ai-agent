# 配额中心 — 多角色使用场景与测试用例

## 角色定义

| 角色                               | 说明                                                | 配额中心模式 |
| ---------------------------------- | --------------------------------------------------- | ------------ |
| **凭据提供者**（Platform Admin）   | 平台管理员，拥有全维度配额管理权限                  | `admin`      |
| **团队管理员**（Team Admin/Owner） | 团队所有者/管理员，可管理团队内所有配额             | `admin`      |
| **团队成员**（Team Member）        | 普通成员，仅可自助设置「本人 + 本人凭据」的平台配额 | `member`     |

## 配额层级说明

| 层级                       | 含义                              | 典型场景                                           |
| -------------------------- | --------------------------------- | -------------------------------------------------- |
| **平台配额**（platform）   | 限制本团队在 Gateway 上的消费护栏 | 全团队每月 $1000；某成员每日 $50；某 Key 每月 $200 |
| **上游配额**（upstream）   | 限制上游凭据调用额度（厂商侧）    | OpenAI 凭据每月 $5000；Azure 凭据每日 $1000        |
| **下游权益**（downstream） | 为虚拟 Key 配置下游客户权益套餐桶 | VIP Key 每月 $500 + 1M Token                       |

## 场景一：凭据提供者（Platform Admin）

### S1-1 首次为团队设置全团队消费护栏

**目标**：限制整个团队每月消费不超过 $2000

**操作路径**：

1. 进入配额中心 → 点击「批量设置」
2. Step 1：层级选「平台配额」→ 主体选「全团队」
3. Step 2：周期选「每月」→ 填写 USD 限额 `2000` → 点「下一步」
4. Step 3：预览 1 条规则 → 确认提交

**预期结果**：成功创建 1 条 `platform / tenant / monthly / $2000` 规则

**潜在问题**：

- [ ] **P1** 「全团队」主体选中后，Step 1 无凭据选择区域——正确，但用户可能困惑"为什么没有凭据选项"。建议：在「全团队」选中时显示一行提示「全团队护栏不区分凭据，如需按凭据限制请选择指定成员」。
- [ ] **P2** 预览表中「主体」列显示「全团队」——OK，但「凭据」列显示「—」，可能让用户误以为没配好。建议：改为「全部」或灰显。

### S1-2 为指定成员批量设置配额

**目标**：为 3 个成员设置每月 $100 的消费限额

**操作路径**：

1. 点击「批量设置」
2. Step 1：层级「平台配额」→ 主体「指定成员」→ 勾选 3 个成员 → 不选凭据 → 下一步
3. Step 2：周期「每月」→ USD `100` → 下一步
4. Step 3：预览 3 条规则 → 提交

**预期结果**：3 条 `platform / user:{id} / monthly / $100` 规则

**潜在问题**：

- [ ] **P3** 成员列表如果很长（50+），搜索+全选体验尚可，但无法按角色筛选（如只选 admin）。建议：可选的「按角色筛选」下拉。
- [ ] **P4** 选了成员后，凭据区域标题写「凭据（可选）」——但用户可能不理解"选了凭据会怎样"。建议：更明确的提示「不选 = 成员总用量护栏；选凭据 = 仅限制该成员在指定凭据上的用量（笛卡尔积）」。

### S1-3 为成员+凭据设置细粒度配额

**目标**：成员 Alice 在 OpenAI 凭据上每月最多 $50

**操作路径**：

1. 点击「批量设置」
2. Step 1：层级「平台配额」→ 主体「指定成员」→ 选 Alice → 选 OpenAI 凭据 → 下一步
3. Step 2：周期「每月」→ USD `50` → 下一步
4. Step 3：预览 1 条规则 → 提交

**预期结果**：1 条 `platform / user:alice / credential:openai / monthly / $50`

**潜在问题**：

- [ ] **P5** 笛卡尔积爆炸风险：2 个成员 × 3 个凭据 × 5 个模型 = 30 条规则，用户可能没意识到。当前有 200 条上限，但预览表只显示前 50 条。建议：在 Step 1 底部实时显示「预计生成 N 条规则」计数器。

### S1-4 为虚拟 Key 设置配额

**目标**：VIP Key 每月 $500

**操作路径**：

1. 点击「批量设置」
2. Step 1：层级「平台配额」→ 主体「虚拟 Key」→ 选 VIP Key → 下一步
3. Step 2：周期「每月」→ USD `500` → 下一步
4. Step 3：预览 → 提交

**预期结果**：1 条 `platform / key:vip / monthly / $500`

### S1-5 为上游凭据设置额度

**目标**：OpenAI 凭据每月最多 $5000

**操作路径**：

1. 点击「批量设置」
2. Step 1：层级「上游配额」→ 不勾「全部凭据」→ 选 OpenAI 凭据 → 下一步
3. Step 2：时间窗口填 `0`（套餐周期）→ USD `5000` → 下一步
4. Step 3：预览 → 提交

**潜在问题**：

- [ ] **P6** 上游配额的「时间窗口（秒）」和「桶标签」对非技术用户非常不友好。0 = 套餐周期是什么意思？建议：提供常用选项下拉（套餐周期 / 每日 / 每月 / 自定义秒数），而非让用户填秒数。
- [ ] **P7** 「桶标签」概念对用户不直观。建议：改为「配额桶名称」并提供说明「同一凭据下不同桶独立计数，默认为 default」。

### S1-6 为下游 Key 设置权益套餐

**目标**：为 3 个 Key 设置每月 $200 + 500K Token 的权益

**操作路径**：

1. 点击「批量设置」
2. Step 1：层级「下游权益」→ 选 3 个 Key → 下一步
3. Step 2：时间窗口 `0` → USD `200` → Token `500000` → 下一步
4. Step 3：预览 3 条 → 提交

**潜在问题**：

- [ ] **P8** 同 S1-6 的时间窗口/桶标签问题。

### S1-7 编辑已有配额规则

**目标**：将某条每月 $100 的规则改为 $200

**操作路径**：

1. 在表格中找到该规则 → 点击编辑按钮（铅笔图标）
2. 进入向导编辑模式 → Step 1 层级/主体锁定不可改 → 下一步
3. Step 2 修改 USD 为 `200` → 下一步
4. Step 3 预览 → 提交

**潜在问题**：

- [ ] **P9** 编辑模式下 Step 1 显示了锁定的层级/主体但不可操作，用户需要点「下一步」跳过——这一步没有实际操作价值。建议：编辑模式直接跳到 Step 2，或 Step 1 只读展示维度信息（不占完整步骤）。
- [ ] **P10** 编辑模式下如果用户想改层级/主体，提示「请删除后新建」——但没有提供直接跳转到删除的入口。建议：在提示旁加「删除此规则」按钮。

### S1-8 删除配额规则

**目标**：删除某条不再需要的配额

**操作路径**：

1. 在表格中找到该规则 → 点击删除按钮（垃圾桶图标）
2. 直接删除（当前无确认对话框）

**潜在问题**：

- [ ] **P11** 删除操作无确认对话框，误点即删。建议：添加确认弹窗或 undo toast。

### S1-9 查看计划类配额（upstream/downstream 来自套餐的配额）

**目标**：了解某条 upstream 配额的来源

**操作路径**：

1. 在表格中看到该规则 → 操作列显示「去凭据页管理」链接
2. 点击跳转到凭据页

**潜在问题**：

- [ ] **P12** 计划类配额在表格中显示但不可编辑/删除，操作列只有链接——用户可能困惑"为什么有些规则能编辑有些不能"。建议：在表格中增加一列「来源」标签（计划/自定义），让区分更直观。

## 场景二：团队管理员（Team Admin）

### S2-1 查看团队整体配额状况

**目标**：了解团队当前有多少条配额、哪些即将超限

**操作路径**：

1. 进入配额中心 → 查看 KPI 概览卡片（规则总数、告警数、平台层用量合计、即将重置）
2. 通过层级/模型/周期筛选器缩小范围
3. 切换卡片视图查看详情

**预期结果**：概览卡片正确展示统计数据

**潜在问题**：

- [x] **P13（已修复）** KPI 金额合计现仅统计 **platform 层**（标题改为「平台层用量合计 / 平台层已用」），消除了 upstream/downstream 跨层重复计数。注：platform 层内 tenant 与 user 规则仍可能因包含关系部分重叠，属护栏总览语义；精确账单见统计页。

### S2-2 为新加入成员设置配额

**目标**：新成员 Bob 加入，为他设置每日 $20 的限额

**操作路径**：

1. 点击「批量设置」→ Step 1 选 Bob → 下一步
2. Step 2 设置每日 $20 → 下一步 → 提交

**潜在问题**：

- [ ] **P14** 如果成员刚加入，members 列表可能还没刷新。建议：成员列表旁加刷新按钮或自动 staleTime 缩短。

### S2-3 批量为所有成员设置统一配额

**目标**：为团队 20 个成员每人每月 $50

**操作路径**：

1. 点击「批量设置」→ Step 1 选「指定成员」→ 全选 → 下一步
2. Step 2 每月 $50 → 下一步
3. Step 3 预览 20 条 → 提交

**预期结果**：20 条规则

**潜在问题**：

- [ ] **P15** 全选后如果成员列表很长，预览表只显示前 50 条，用户无法逐一确认。建议：预览表增加汇总行（如「共 20 条，全部为每月 $50」）。

### S2-4 管理上游凭据额度

**目标**：查看并调整 OpenAI 凭据的上游额度

**操作路径**：

1. 筛选层级「上游配额」→ 找到 OpenAI 凭据的规则
2. 点击编辑 → 修改限额

**潜在问题**：

- [ ] **P16** 上游配额如果是计划类（来自套餐），操作列显示「去凭据页管理」——跳转后用户到了凭据列表页，还需要再找对应凭据。建议：链接带上 credential_id 参数，直接定位到该凭据详情。

## 场景三：团队成员（Team Member）

### S3-1 首次设置自助配额

**目标**：成员 Alice 想给自己设一个每月 $30 的消费上限，防止超支

**操作路径**：

1. 进入「我的配额」→ 点击「设置我的配额」
2. Step 1：选择凭据（看到自己创建的 + legacy 团队凭据）→ 勾选 2 个凭据 → 下一步
3. Step 2：每月 $30 → 下一步
4. Step 3：预览 2 条规则 → 提交

**预期结果**：2 条 `platform / user:alice / credential:{id} / monthly / $30`

**潜在问题**：

- [ ] **P17** Legacy 凭据标注了「团队」Badge，但用户可能不理解这是什么意思。建议：hover 时显示 tooltip「团队共享凭据（由管理员创建），您可为其设置自助限额」。
- [ ] **P18** 成员模式 Step 1 只显示凭据选择，没有层级/主体选择——但 Step 指示器仍显示「选择凭据」。如果用户只有 BYOK 凭据（在凭据页管理），这里列表为空，体验很差。建议：空列表时显示更明确的引导（如「您当前没有可设置配额的团队凭据。个人 BYOK 凭据请在凭据页设限。」）——当前已有 emptyHint，但可以更突出。

### S3-2 从凭据详情页跳转设置配额

**目标**：Alice 在凭据页看到自己的凭据，点击「设置我的限额 →」

**操作路径**：

1. 凭据详情页 → 点击「设置我的限额 →」
2. 自动跳转到配额中心，URL 带 `?credential_id=xxx`
3. 向导自动打开，凭据预填
4. Step 2 设置限额 → Step 3 预览 → 提交

**预期结果**：凭据预填成功，直接进入 Step 2

**潜在问题**：

- [ ] **P19** URL 参数 `credential_id` 预填依赖 `credentialOptions` 加载完成。如果凭据列表加载慢，预填可能失败（consumedPrefillRef 机制只在首次匹配时触发）。建议：增加 loading 状态提示。

### S3-3 修改已有的自助配额

**目标**：Alice 把每月 $30 改为 $50

**操作路径**：

1. 在「我的配额」列表找到该规则 → 点击编辑
2. 向导编辑模式 → Step 2 修改 USD → 提交

**预期结果**：限额更新为 $50

### S3-4 删除自助配额

**目标**：Alice 不想再限制自己

**操作路径**：

1. 找到该规则 → 点击删除

**预期结果**：规则被删除

**潜在问题**：

- [ ] **P20** 同 S1-11，删除无确认。

### S3-5 查看配额使用情况

**目标**：Alice 想知道自己这个月用了多少

**操作路径**：

1. 在列表/卡片中查看 usage 列/进度条

**预期结果**：能看到当前用量和占比

**潜在问题**：

- [ ] **P21** 成员模式下，概览 KPI 卡片展示的是所有可见规则的汇总——但成员只能看到自己的规则，KPI 的「告警配额」对个人用户意义不大。建议：成员模式的 KPI 简化为「我的配额数 / 已用额度 / 即将重置」。

### S3-6 Legacy 凭据配额设置

**目标**：Alice 想为团队共享的 legacy 凭据设置自助限额

**操作路径**：

1. Step 1：凭据列表中看到标注「团队」的 legacy 凭据 → 勾选
2. Step 2：设置限额 → 提交

**预期结果**：成功创建自助配额（仅约束 Alice 本人在该凭据上的消费）

**潜在问题**：

- [ ] **P22** Legacy 凭据的「团队」Badge 可能让用户犹豫"这是不是只有管理员才能操作？"。建议：Badge 改为「共享」更友好，tooltip 说明「团队共享凭据，您可为其设置自助限额」。

## 测试用例清单

### TC-01 管理员 — 全团队护栏

1. 以管理员身份进入配额中心
2. 点击「批量设置」
3. Step 1：层级=平台配额，主体=全团队
4. Step 2：周期=每月，USD=2000
5. Step 3：确认预览 1 条规则，提交
