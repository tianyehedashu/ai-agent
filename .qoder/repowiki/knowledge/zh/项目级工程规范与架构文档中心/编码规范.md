- 列表 API 必须采用基于 `page`/`page_size` 的分页信封（Envelope）响应格式，禁止直接返回裸数组或使用 `skip/limit` 作为对外查询参数。
- 管理面 REST API 的成功响应应直接输出资源 Schema（直出模式），错误响应统一遵循 RFC 7807 Problem Details 标准，禁止使用 `{ code, data }` 包裹结构。
- SSO 集成采用网关侧身份桥接模式，后端仅解析 HiGress 注入的 `X-Giikin-*` 请求头并进行 JIT 用户 provisioning，严禁后端服务直连 IAM Redis。
- 日志系统实行前后端统一规范，后端通过 `StructuredLogger` 记录结构化事件（如 LLM 调用、MCP 工具执行），前端集成 Sentry 进行错误追踪。
