- 各业务域（domains）内部统一采用 Presentation（路由/Schema）、Application（UseCase/Service）、Domain（实体/接口）、Infrastructure（仓储/ORM）的分层组织方式。
- 数据库会话通过 `libs.db.database.get_db` 提供，并在 `libs/api/deps.py` 中定义为 `DbSession` 依赖，确保事务在请求结束时根据是否有写入操作自动提交或回滚。
- 权限校验逻辑下沉至 `libs/iam` 层，通过 `PermissionContext` 和 `assert_tenant_access` 等纯函数实现，避免在表现层重复编写鉴权代码。
- 异常处理遵循 RFC 7807 Problem Details 规范，在 `bootstrap/main.py` 中统一定义全局异常处理器，将领域错误映射为标准的 JSON 响应格式。
