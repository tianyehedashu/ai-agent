- 所有 API 响应遵循 RFC 7807 Problem Details 规范，通过 `libs/api/problem_details.py` 统一转换领域异常为标准化 JSON 错误体。
- 数据库会话依赖注入采用读写分离策略：在事务退出时自动检测 ORM 写入标记，有变更则提交，纯读请求则回滚以释放连接。
- 领域间通信通过 ASGI 中间件（如 `PermissionContextASGIMiddleware`）或应用级背景任务队列实现，避免循环导入并保持表现层轻量化。
- 异步资源清理（如 LiteLLM 客户端、Redis 连接）统一收口在 `bootstrap/main.py` 的 `lifespan` 关闭阶段，确保服务停止时的资源确定性释放。
